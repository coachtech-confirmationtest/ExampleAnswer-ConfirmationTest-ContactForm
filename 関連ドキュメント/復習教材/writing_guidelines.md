# 復習教材 執筆ガイドライン

## 1. 基本構成（学習の流れを意識したテンプレート）

全ての応用機能Chapterは、以下の構成を基本とします。この流れは、学習者が「なぜ(Why) → どうやって(How) → 何を(What)」の順で自然に理解を深められるように設計されています。

| No. | セクション名 | 目的と内容 |
|-----|------------|-----------|
| 1 | はじめに | このChapterで何を学ぶのか、それがなぜ重要なのかを提示し、学習のゴールを明確にする。 |
| 2 | 要件の確認 | 実装する機能の具体的な仕様を表形式で整理する。ゴールが明確になることで、実装のブレを防ぐ。 |
| 3 | 先輩エンジニアの思考プロセス | **【最重要】** 実装コードを見る前に、「なぜこの実装になるのか」という設計段階の思考を4〜5つのポイントで解説する。 |
| 4 | 実装 | **【省略厳禁】** 実装に必要な全てのコード（ルート、コントローラー、モデル、リソース等）を完全な形で掲載する。 |
| 5 | コードの詳細解説 | 実装コードを論理的なブロックに分け、各ブロックの役割と各行の構文を丁寧に解説する。 |
| 6 | How to: この実装にたどり着くための調べ方 | 初心者が自力でこの実装にたどり着くための、実践的な検索キーワードとプロセスを段階的に示す。 |
| 7 | (あれば) 提供されているBladeファイルの確認 | ビュー側の実装で重要なポイントがあれば解説する。 |
| 8 | まとめ | このChapterで学んだ重要な概念やテクニックを箇条書きで振り返る。 |

## 2. 各セクションの執筆ポイント

### 2.1. 先輩エンジニアの思考プロセス

- **目的**: コードの「正解」をいきなり見せるのではなく、その「正解」に至るまでの思考の過程を追体験させる。
- **書き方**: 4〜5つの「思考（問い）」を立て、それに対する「答え（結論と理由）」を先輩エンジニアが語りかける形式で記述する。
- **例**: 「どうやって効率的にデータを取得するか？」「APIのエラーレスポンスをどうやって統一するか？」

### 2.2. 実装

- **ルール**: コードや手順は絶対に省略しない。 「// ...（省略）」のような記述は一切使用しない。
- **理由**: 学習者がコードをコピー＆ペーストした際に、省略箇所が原因でエラーになることを防ぎ、教材の信頼性を担保する。

### 2.3. コードの詳細解説

- **基本パターン**: テストコードの解説では「AAA（Arrange, Act, Assert）」パターンを明記し、コードの構造を意識させる。
- **構文解説**: アロー関数、メソッドチェーン、data_get()ヘルパなど、初学者がつまずきやすい構文は、コードを抜き出して番号付きリストで丁寧に解説する。
- **比較**: assertSame vs assertEquals、assertHeader vs assertHeaderContainsなど、似て非なるメソッドの違いを明確に解説する。
- **なぜ**: 「なぜこのメソッドを使うのか」「なぜこの実装が優れているのか」（例: N+1問題の解決、DRY原則）という理由を必ず添える。

### 2.4. How to: この実装にたどり着くための調べ方

- **目的**: 「答え」を教えるだけでなく、「答えの見つけ方」を教える。
- **NG例**: `laravel query builder where or` のような、答えを知らないと検索できないキーワードを提示する。
- **OK例**: 
  - Step 1: 「Laravel 検索機能 複数条件」で検索 
  - → Step 2: 見つかった記事から「クロージャを使ったwhere句のグループ化」という概念を知る 
  - → Step 3: 「Laravel where クロージャ」でさらに検索 
  
  のように、思考の変遷がわかる段階的なプロセスを示す。

## 3. 全体を通しての心構え

- **学習者の視点に立つ**: 常に「自分は今、初めてこの概念を学ぶ」という視点を持ち、専門用語を無意識に使わない。
- **一貫性を保つ**: 全てのChapterで同じ構成と執筆スタイルを維持し、学習者が安心して読み進められるようにする。
- **信頼性を担保する**: コードやコマンドは全て検証済みであり、省略がないことを保証する。エラーハンドリングやエッジケースの考慮も忘れない。
- **「なぜ」を繰り返す**: 全てのコード、全ての設計判断に対して「なぜそうするのか？」という問いを立て、その答えを平易な言葉で説明する。
